# Product

## Register

product

## Users

Portfolio and demo project visitors. Developers, designers, and recruiters evaluating technical skill through a polished auth experience. Context: browsing a personal portfolio site, evaluating craftsmanship.

## Product Purpose

Demonstrate production-grade frontend skills: authentication flows (register, login, logout, current user), WebGL visual effects, smooth animations, and clean code architecture. Serves as a technical showcase where the auth UI itself is the artifact.

## Brand Personality

Playful, dynamic, expressive. The interface should feel alive: animated backgrounds, micro-interactions on every touchpoint, motion that rewards attention. Personality emerges through interaction, not decoration.

## Anti-references

Generic auth templates with bland cards, stock login forms, and zero personality. No flat Material Design login boxes, no Bootstrap card components, no skeuomorphic glossy buttons, no overdone cyberpunk neon-everything cliches. The neon palette should feel intentional and restrained, not a full saturation dump.

## Design Principles

1. Motion is first-class: every element has a reason to move, and that motion communicates state or guides attention.
2. WebGL as atmosphere, not gimmick: the 3D background serves the experience, it does not compete with the forms.
3. Interaction rewards curiosity: hover states, focus rings, and transitions should feel satisfying and precise.
4. Auth flows as choreography: register and login are not just forms; they are moments of entry with visual identity.
5. Restraint within expressiveness: neon accents pop because the surrounding canvas is controlled, not because everything glows.

## Accessibility & Inclusion

WCAG AA contrast for all text. Focus-visible indicators on all interactive elements. Reduced motion alternative for WebGL background (static gradient fallback). Keyboard-navigable forms with clear focus order. Semantic HTML for screen readers.
